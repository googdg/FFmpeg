# AWS活动管理系统 - 快速启动指南

## 🚀 5分钟快速上手

### 第一步：启动系统
```bash
cd aws-event-manager
./start.sh
```

看到以下信息表示启动成功：
```
🚀 AWS活动管理系统启动成功！
📍 管理后台: http://localhost:3000
📍 报名页面: http://localhost:3000/registration.html
```

### 第二步：验证系统
```bash
./test-simple.sh
```

看到 `🎉 所有核心功能测试通过！` 表示系统正常。

### 第三步：开始使用

#### 管理员操作
1. 访问 http://localhost:3000
2. 使用账号 `admin` / `admin123` 登录
3. 创建第一个活动
4. 管理报名和签到

#### 用户报名
1. 访问 http://localhost:3000/registration.html
2. 选择活动并填写报名信息
3. 提交报名

---

## 📋 常用操作

### 创建活动
1. 登录管理后台
2. 点击"创建活动"
3. 填写活动信息：
   - 活动标题
   - 日期时间
   - 地点
   - 讲师信息
   - 最大人数
4. 点击"创建活动"

### 管理报名
1. 在活动列表中点击"查看报名"
2. 查看报名人员信息
3. 可以删除或修改报名状态

### 现场签到
1. 在报名列表中点击"签到"按钮
2. 系统自动记录签到时间
3. 查看签到统计

### 上传照片
1. 点击活动的"管理照片"
2. 选择照片文件上传
3. 照片会显示在活动详情中

## 🔧 故障排除

### 启动失败
```bash
# 检查端口是否被占用
lsof -i :3000

# 杀死占用进程
kill -9 <PID>

# 重新启动
./start.sh
```

### 功能异常
```bash
# 运行测试脚本
./test-simple.sh

# 查看详细测试
open http://localhost:3000/quick-test.html
```

### 数据库问题
```bash
# 备份数据库
cp events.db events.db.backup

# 重置数据库（会丢失数据）
rm events.db
./start.sh
```

## 📞 技术支持

如果遇到问题：
1. 首先运行 `./test-simple.sh` 检查系统状态
2. 查看控制台错误信息
3. 检查 `events.db` 文件是否存在且可写
4. 确认Node.js版本 >= 14
