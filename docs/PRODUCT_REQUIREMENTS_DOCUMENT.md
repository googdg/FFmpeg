## 🎯 产品概述

### 产品定位
专业的医学科普视频学习平台，采用邀请码验证机制，为特定用户群体提供高质量的医学教育内容。

### 目标用户
- **主要用户**: 医学专业学生、医护人员、医学研究者

- 提供专业、权威的医学科普内容

## 🎯 核心功能需求

### 1. 邀请码验证系统

#### 功能描述
通过邀请码机制控制平台访问权限，确保用户质量。

#### 详细需求
- **邀请码格式**: 任意数字组合（如：1234、888、2024）
- **验证机制**: 会话级验证，关闭浏览器后需重新验证
- **验证流程**: 
  1. 用户首次访问视频时弹出邀请码输入框
  2. 输入任意数字邀请码
  3. 验证成功后可观看所有视频
  4. 关闭浏览器后验证状态自动清除

#### 用户故事
```
作为一个新用户
我想要通过邀请码验证身份
以便获得观看医学视频的权限

验收标准：
- 首次点击视频时显示邀请码输入框
- 输入任意数字后可以通过验证
- 验证成功后当前会话可观看所有视频
- 关闭浏览器后需要重新验证
```

### 2. 视频分类浏览系统

#### 功能描述
按医学专业分类组织视频内容，便于用户快速找到所需内容。

#### 分类体系
1. **🎬 平台演示** - 平台功能介绍和使用指南
2. **🩺 临床医学** - 临床诊疗相关视频
3. **📚 基础医学** - 解剖、生理、病理等基础知识
4. **🔬 外科手术** - 手术技术和操作指导
5. **🫁 内科诊疗** - 内科疾病诊断和治疗
6. **💊 药学治疗** - 药物治疗和用药指导
7. **🤖 前沿技术** - 医学新技术和创新应用
8. **⚖️ 伦理法规** - 医学伦理和法律法规

#### 用户故事
```
作为一个医学学习者
我想要按照专业分类浏览视频
以便快速找到我需要的学习内容

验收标准：
- 显示清晰的分类标签
- 点击分类可筛选对应视频
- 支持"全部"分类查看所有视频
- 分类切换响应迅速
```

### 3. 视频播放系统

#### 功能描述
提供流畅的视频播放体验，支持多种播放控制功能。

#### 详细需求
- **播放界面**: 专门的视频详情页面
- **播放控制**: 播放/暂停、进度控制、音量调节
- **视频信息**: 标题、描述、讲师信息、时长
- **响应式设计**: 适配桌面端和移动端

#### 用户故事
```
作为一个学习者
我想要在专门的页面观看视频
以便获得更好的学习体验

验收标准：
- 点击视频卡片跳转到视频详情页
- 视频可以正常播放和控制
- 显示完整的视频信息
- 移动端播放体验良好
```

### 4. 响应式用户界面

#### 功能描述
提供现代化的YouTube风格界面，支持多设备访问。

#### 设计要求
- **设计风格**: 仿YouTube的现代化界面
- **颜色方案**: 白色背景，蓝色主色调
- **布局方式**: 网格布局，响应式设计
- **交互效果**: 流畅的动画和过渡效果

#### 用户故事
```
作为一个用户
我想要在任何设备上都能舒适地使用平台
以便随时随地进行学习

验收标准：
- 桌面端显示多列网格布局
- 移动端自适应为单列布局
- 触摸操作响应良好
- 界面美观现代
```

## 🎨 用户体验需求

### 界面设计原则
1. **简洁明了**: 界面简洁，信息层次清晰
2. **一致性**: 保持设计风格和交互模式一致
3. **可访问性**: 支持不同能力用户的访问需求
4. **响应性**: 快速响应用户操作

### 交互设计要求
1. **直观操作**: 用户无需学习即可上手使用
2. **反馈及时**: 操作后立即给出视觉反馈
3. **容错性**: 提供清晰的错误提示和恢复方式
4. **效率优化**: 减少用户完成任务的步骤

- **辅助色**: #7b68ee (紫色)
- **背景色**: #ffffff (白色)
- **文字色**: #333333 (深灰)
- **边框色**: #e0e0e0 (浅灰)
