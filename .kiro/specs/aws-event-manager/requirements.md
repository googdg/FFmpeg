# AWS开发者社区活动管理系统 - 需求文档

## 项目概述

### 项目名称
AWS开发者社区活动管理系统 (AWS Event Manager)

### 项目目标
为AWS开发者社区提供一个完整的H5活动管理系统，支持活动策划、宣传、报名、签到和总结的全流程管理。

### 目标用户
- **主要用户**: AWS社区组织者 (Stevn)
- **次要用户**: 开发者参会者
- **管理员**: 社区管理团队

## 核心功能需求

### 1. 活动管理模块

#### 1.1 活动创建
**用户故事**: 作为社区组织者，我希望能够创建新活动，包含完整的活动信息。

**功能要求**:
- 活动基本信息录入
  - 活动主题 (必填)
  - 活动时间 (日期+时间)
  - 活动地址 (详细地址)
  - 活动介绍 (富文本)
  - 预计人数
- 嘉宾信息管理
  - 嘉宾姓名、职位、公司
  - 嘉宾头像上传
  - 嘉宾介绍
- 活动状态管理
  - 草稿、已发布、进行中、已结束

**验收标准**:
- ✅ 能够成功创建活动并保存
- ✅ 必填字段验证正确
- ✅ 活动信息可以编辑和更新
- ✅ 支持图片上传功能

#### 1.2 活动列表
**用户故事**: 作为组织者，我希望能够查看所有活动的列表和状态。

**功能要求**:
- 活动列表展示
- 按状态筛选 (全部/草稿/已发布/进行中/已结束)
- 按时间排序
- 快速操作 (编辑/删除/发布)

**验收标准**:
- ✅ 活动列表正确显示
- ✅ 筛选功能正常工作
- ✅ 操作按钮功能正确

### 2. 报名管理模块

#### 2.1 报名表单
**用户故事**: 作为开发者，我希望能够方便地报名参加感兴趣的活动。

**功能要求**:
- 报名表单字段
  - 姓名 (必填)
  - 公司 (必填)
  - 岗位 (必填)
  - 年龄 (可选)
  - 手机号 (必填)
  - 邮箱 (可选)
  - 备注 (可选)
- 表单验证
- 重复报名检测
- 报名成功确认

**验收标准**:
- ✅ 表单验证正确
- ✅ 能够成功提交报名
- ✅ 防止重复报名
- ✅ 显示报名成功状态

#### 2.2 报名管理
**用户故事**: 作为组织者，我希望能够查看和管理所有报名信息。

**功能要求**:
- 报名列表查看
- 报名统计 (总数/已签到/未签到)
- 报名信息导出 (CSV)
- 报名状态管理

**验收标准**:
- ✅ 报名列表正确显示
- ✅ 统计数据准确
- ✅ 导出功能正常

### 3. 签到管理模块

#### 3.1 现场签到
**用户故事**: 作为组织者，我希望能够在活动现场快速为参会者签到。

**功能要求**:
- 签到界面
  - 手机号快速查找
  - 姓名模糊搜索
  - 一键签到
- 签到状态显示
- 签到时间记录
- 现场统计

**验收标准**:
- ✅ 搜索功能快速准确
- ✅ 签到操作简单流畅
- ✅ 状态更新及时
- ✅ 统计数据实时更新

### 4. 活动总结模块

#### 4.1 照片上传
**用户故事**: 作为组织者，我希望能够上传活动照片并管理。

**功能要求**:
- 批量照片上传
- 照片预览和管理
- 照片描述添加
- 照片删除功能

**验收标准**:
- ✅ 支持多张照片上传
- ✅ 照片正确显示
- ✅ 管理功能正常

#### 4.2 活动总结
**用户故事**: 作为组织者，我希望能够撰写和发布活动总结。

**功能要求**:
- 总结内容编辑 (富文本)
- 活动数据统计
- 总结发布和分享
- 总结模板

**验收标准**:
- ✅ 编辑器功能完整
- ✅ 数据统计准确
- ✅ 发布功能正常

## 技术需求

### 前端技术栈
- HTML5 + CSS3 + ES6+ JavaScript
- 响应式设计 (移动端优先)
- PWA支持 (离线功能)
- 图片压缩和优化

### 后端技术栈
- Node.js + Express.js
- SQLite数据库
- JWT身份认证
- 文件上传处理

### 数据存储
- 活动信息
- 用户报名数据
- 签到记录
- 照片和文件

## 非功能性需求

### 性能要求
- 页面加载时间 < 3秒
- 支持100+并发用户
- 图片自动压缩优化

### 安全要求
- 数据输入验证
- XSS防护
- 文件上传安全检查

### 可用性要求
- 移动端友好
- 直观的用户界面
- 操作流程简单

### 兼容性要求
- 支持主流移动浏览器
- iOS Safari 12+
- Android Chrome 70+

## 用户流程

### 组织者流程
1. 登录系统
2. 创建新活动
3. 发布活动
4. 查看报名情况
5. 现场签到管理
6. 上传照片
7. 撰写总结

### 参会者流程
1. 查看活动详情
2. 填写报名表单
3. 收到报名确认
4. 参加活动
5. 现场签到
6. 查看活动总结

## 数据模型

### 活动 (Events)
- id, title, description, date, time, location
- speaker_info, max_attendees, status
- created_at, updated_at

### 报名 (Registrations)
- id, event_id, name, company, position
- age, phone, email, notes
- status, registered_at

### 签到 (CheckIns)
- id, registration_id, checked_in_at
- checked_by

### 照片 (Photos)
- id, event_id, filename, description
- uploaded_at

### 总结 (Summaries)
- id, event_id, content, published_at
